### 1. What is ERC-8004 and Its Purpose

ERC-8004 is an Ethereum protocol standard designed to create a **minimal trust layer for automated services** called trustless agents.[1] The core mission is to "use blockchains to discover, choose, and interact with agents across organizational boundaries without pre-existing trust, thus enabling open-ended agent economies."[1]

The fundamental problem ERC-8004 solves relates to **fragmentation and isolated registries** in autonomous agent systems.[3] Without standardization, agents operate in silos with incompatible frameworks and no means of establishing credible reputation across different platforms. ERC-8004 provides a neutral, on-chain infrastructure that enables agents to:

- **Establish portable identity**: Each agent receives a unique, globally-recognized identity anchored on-chain
- **Build verifiable reputation**: Create auditable records of successful interactions and task completion
- **Enable trustless discovery**: Allow other agents, marketplaces, and users to find and evaluate agents based on standardized criteria
- **Facilitate cross-organizational collaboration**: Enable agents from different organizations to interact without intermediaries

The standard focuses specifically on **who** an agent is and **why** a counterparty might trust its outputs, leaving implementation details of agent capabilities and message exchange formats flexible and extensible.[1]

### 2. Technical Specifications and Standards

### Core Registry Architecture

ERC-8004 implements three interconnected, on-chain registries that form the backbone of agent trust infrastructure:[3]

**Identity Registry**

The Identity Registry assigns each agent a globally unique, tokenized entry using ERC-721 semantics (NFT standards).[1] Each agent receives a portable on-chain identity represented as an NFT, which serves as a machine identity token.[2][4] This token points to an off-chain agent registration file stored via URI, containing:

- Agent name and description
- Capabilities and supported skills
- Associated wallet address
- Agent-to-Agent (A2A) or Model Context Protocol (MCP) endpoints
- Supported protocols and domains

Since the identity entry is standardized and based on neutral infrastructure, any marketplace or browser can index and discover agents using this common reference.[2][4]

**Reputation Registry**

The Reputation Registry creates an on-chain audit trail of client-authorized feedback, enabling reputation tracking without storing all data on-chain.[3] This registry:

- Allows agents to leave structured feedback for each other, tagged by specific task
- Links feedback to **x402** (EIP-402) cryptographic receipts that tie on-chain payments to off-chain interactions[2][4]
- Maintains a verifiable history of agent performance across different work categories
- Enables reputation scoring without requiring complete data storage on-chain, optimizing for scalability

**Validation Registry**

The Validation Registry records verification outcomes and supports task verification through multiple mechanisms:[3] These include crypto-economic staking, cryptographic proofs using Trusted Execution Environments (TEEs), or zero-knowledge machine learning (zkML) verification.[2][3][4] For higher-trust use cases, validators can confirm outputs through:

- Hardware-isolated enclaves (TEEs) for confidential execution
- Proof-of-stake mechanisms
- Zero-knowledge proofs (zkML)

ERC-8004 follows a **composable, minimal design** philosophy:[1] The standard anchors identity and compact trust signals on-chain through the three registries while keeping large or evolving data off-chain via URIs.
